How to achieve an agile software architecture ของ Martin Fowler จะเน้นไปที่ วิธีการที่ทำให้สถาปัตยกรรมมีความ flex หรือยืดหยุ่น คือ ไม่ต้องออกแบบทุกอย่างให้ถูก 100% แต่ต้องออกแบบให้ระบบ สามารถเปลี่ยนแปลง ได้ง่าย แล้วมีการพูดถึง agile ว่าเกิดจาก CI/CD ที่ทำให้มีการแก้ไขโค้ดและ deploy บ่อยๆ
ปรับปรุงโครงสร้างโค้ดตลอดเวลา ไม่ใช่รอทำทีเดียวตอนท้าย
 testing ที่ทำให้เรามั่นใจว่าการแก้โค้ด ไม่ได้ทำให้พังทั้งระบบ
ใช้ vertical slices คือ ไม่สร้างเแบบ layer แต่สร้างทีละละอันแบบ n2n  เพื่อให้เห็นภาพรวมและ feedback ได้เร็ว
ไม่รีบตัดสินใจ เพื่อให้เรามีข้อมูลมากๆ
